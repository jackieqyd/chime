# 舌尖账本 - 产品需求文档

> 版本：v1.0 | 状态：草稿 | 更新日期：2026-04-29

---

## 一、产品概述

### 1.1 产品定位

**舌尖账本**是一款面向健康饮食管理的移动应用（APP + 微信小程序），通过记录用户日常饮食摄入，提供个性化热量分析和反馈，帮助用户建立健康饮食习惯。

产品名称由"吃了么"演进而来，更强调**记录**与**分析**的功能定位。

### 1.2 核心价值

- **简单快捷的食物记录** — 拍照留存、搜索添加、手动补充多种方式并存
- **智能个性化的饮食反馈** — 基于记录数据提供热量分析和健康建议

### 1.3 目标用户

| 属性 | 描述 |
|------|------|
| 年龄段 | 18-35岁 |
| 用户特征 | 关注健康、需要热量管理 |
| 典型场景 | 用餐后记录、查看历史饮食、热量管理 |

### 1.4 双版本模式

产品提供**自律版**与**嘴馋版**两种使用模式，满足不同用户心理需求：

| 维度 | 自律版 | 嘴馋版 |
|------|--------|--------|
| 界面风格 | 简洁专业、克制内敛 | 轻松活泼、色彩丰富 |
| 交互设计 | 高效直接、功能导向 | 游戏化引导、鼓励式反馈 |
| 数据展示 | 详细图表、理性分析 | 简化呈现、正向激励 |
| 报告维度 | 热量、营养素、趋势分析 | 达成率、徽章、阶段性成果 |

## 二、功能需求

### 2.1 食物记录（核心功能）

#### 2.1.1 拍照记录

- 用户拍摄餐食照片，保存为饮食记录凭证
- 照片与记录关联展示，支持相册选择或实时拍摄
- 照片水印：记录时间、简要备注
- **离线缓存**：拍照后的照片默认先本地保存，以备后续AI识别功能使用

#### 2.1.2 搜索添加

- 内置食物库（已有数据），按食物分类组织，支持关键词搜索
- 食物库内容：分类层级 + 每种食物的营养成分（热量、碳水、蛋白质、铁、钠等）
- 搜索结果展示：食物名称、所属分类、热量/100g、常见分量
- 支持收藏常用食物到"我的常吃"
- **库中无此食物时**：引导用户手动录入营养信息，或直接拍照记录（照片暂存供后续AI识别）

#### 2.1.3 手动补充

- 自定义食物名称和各营养成分数值（热量、碳水、蛋白质、铁、钠等）
- 支持多食物组合一次性添加（同一餐多个食物）
- 记录用餐时间（默认当前时间，可修改）

#### 2.1.4 记录结构

```
每条饮食记录：
├── 记录ID
├── 用户ID
├── 记录时间
├── 餐次类型（早餐/午餐/晚餐/加餐）
├── 食物列表
│   ├── 食物名称
│   ├── 食物重量/g
│   ├── 热量/kcal
│   └── 营养素信息（可选）
├── 照片（可选）
└── 备注（可选）
```

### 2.2 热量分析

#### 2.2.1 今日热量概览

- 当日已摄入热量
- 今日目标热量（用户设置）
- 剩余可摄入热量
- 热量来源分布（早/午/晚/加餐占比）
- **价格统计**：当日食物消费总额

#### 2.2.2 历史记录查看

- 日/周/月视图切换
- 每日饮食清单回顾
- 支持按日期检索

#### 2.2.3 热量趋势分析（自律版特色）

- 7日/30日热量曲线图
- 周均值对比
- 峰值/低谷标注及原因追溯

### 2.3 个性化反馈

#### 2.3.1 饮食评价

- 基于当日摄入情况给出简评（过多/适中/偏少）
- 营养结构建议（蛋白质/碳水/脂肪比例提示）

#### 2.3.2 激励反馈（嘴馋版特色）

- 达成每日目标时展示激励动画/徽章
- 连续达成目标天数累计展示
- 正向语言鼓励替代冰冷数据

### 2.4 用户设置

- **目标设置**：每日热量目标、体重目标
- **版本切换**：随时可在自律版/嘴馋版之间切换
- **饮食偏好**：忌口食物、过敏原标注（记录时提示）
- **提醒设置**：用餐提醒时间推送

## 三、非功能需求

### 3.1 性能要求

- 食物搜索响应时间 < 500ms
- 页面切换流畅，无明显卡顿
- 照片压缩上传，节省流量

### 3.2 数据安全

- 用户饮食数据仅用户本人可见
- 支持数据导出（Excel/CSV格式）
- 隐私政策合规

### 3.3 兼容性

- APP：iOS 12+、Android 8.0+
- 小程序：微信版本兼容

## 五、后续规划

- **AI拍照识别**：餐食照片AI自动识别食物种类并估算热量
- **社交分享**：饮食记录分享至社交平台如微信朋友圈或微信好友等。
- **营养师咨询**：对接专业营养师一对一服务
