# 第一部分：设计规范

## 1.1 设计原则

### 1.1.1 核心理念

**自律版（舌尖账本）**
- 简约克制：专注于核心功能，去除干扰
- 专业可信：通过数据可视化建立信任感
- 温暖关怀：鼓励式设计，不制造焦虑

**欢乐版**
- 活泼有趣：用游戏化元素保持用户兴趣
- 即时满足：快速正反馈，激励持续使用
- 社交激励：社区、挑战、成就系统

### 1.1.2 设计原则

| 原则 | 说明 |
|------|------|
| 清晰优先 | 信息层次分明，用户能快速找到目标 |
| 操作便捷 | 核心操作3步内完成 |
| 反馈明确 | 每个操作都有即时反馈 |
| 容错设计 | 防止误操作，提供撤销/恢复 |
| 一致性 | 同类操作保持一致的行为模式 |

### 1.1.3 设计哲学

- **自律版**：Less is More，功能做减法，体验做加法
- **欢乐版**：Play More，用游戏化思维设计每个交互

### 1.2.4 色彩使用规则

1. **主色使用占比**：主色不超过页面面积的20%
2. **文字与背景对比度**：重要文字对比度 ≥ 4.5:1
3. **功能性色彩**：成功/警告/错误状态严格按规范使用
4. **无障碍考虑**：避免仅依赖颜色传达信息

### 数据计算逻辑

```
每日总摄入（目标）= 根据用户体重自动计算
- 基础计算公式：TDEE = BMR × 活动系数
- 用户更新体重后，目标自动重新计算

当前摄入 = 今日所有食物记录热量之和
- 每添加一条记录，自动累加
- 每删除一条记录，自动减去

百分比 = 当前摄入 / 每日总摄入 × 100%
```

### 水波纹动画规格

```
触发时机：每次添加食物记录后
动画效果：
1. 百分比数字从原值跳动到新值（500ms）
2. 圆环内水波纹向上扩散
3. 水波纹高度 = 百分比值对应的圆环高度
4. 水波纹有3层，最外层最淡

动画参数：
- 水波纹持续时间：800ms
- 水波纹数量：3层
- 数字跳动时长：500ms
- 动画曲线：ease-out
```

```
当日日期判断：

- 未录入（黄色圆点）：当日尚未记录

- 点击日期后，日历下方显示该日记录
```

### 计算公式

```
男BMR = 66.47 + (13.75 × 体重kg) + (5.003 × 身高cm) - (6.755 × 年龄)
女BMR = 655.1 + (9.563 × 体重kg) + (1.85 × 身高cm) - (4.676 × 年龄)

TDEE = BMR × 活动系数
- 久坐: 1.2
- 轻度: 1.375
- 中度: 1.55
- 高度: 1.725
- 极端: 1.9

推荐热量：
- 维持: TDEE
- 减脂: TDEE × 0.8
- 增肌: TDEE × 1.1
```
