> 版本：v1.0 | 状态：待评审 | 更新日期：2026-04-29

## 一、数据库概述

### 1.1 数据库架构

| 数据库 | 用途 | 数据来源 |
|--------|------|----------|
| **QwTasteNote** | 食物库（已有） | 已有数据，直接对接 |
| **Chime** | 业务库（新建） | 用户数据、记录数据 |

### 1.2 数据库设计原则

- **食物库复用**：已有数据库直接对接，不迁移数据，通过EF Core多DbContext配置实现跨库查询
- **业务库独立**：新建业务数据库（Users, FoodRecords等），与食物库分离
- **字段冗余**：FoodRecords表冗余存储食物快照（名称、营养成分），避免历史数据因食物库变更而受影响

## 三、业务数据库（Chime）

> 新建数据库，用于存储用户数据和饮食记录

### 3.1 用户表（Users）

#### 表结构

```sql
CREATE TABLE Users (
    Id                  INT PRIMARY KEY IDENTITY(1,1),

    -- 认证相关
    OpenId              NVARCHAR(100) NULL,       -- 微信OpenId（小程序用户唯一标识）
    UnionId             NVARCHAR(100) NULL,       -- 微信UnionId（同一主体下多端共用）
    AppleId             NVARCHAR(100) NULL,       -- Apple ID（iOS用户标识）
    PhoneNumber         NVARCHAR(20) NULL,        -- 手机号
    PasswordHash        NVARCHAR(200) NULL,      -- 密码哈希（APP用户）

    -- 基础信息
    Nickname            NVARCHAR(50) NULL,
    Avatar              NVARCHAR(200) NULL,
    Gender              TINYINT NULL,             -- 0-女，1-男
    Height              DECIMAL(5,2) NULL,        -- 身高(cm)
    Weight              DECIMAL(5,2) NULL,        -- 体重(kg)
    Age                 INT NULL,

    -- 偏好设置
    VersionMode         TINYINT NOT NULL DEFAULT 0,  -- 0-自律版，1-嘴馋版
    ActivityLevel       TINYINT NOT NULL DEFAULT 1,  -- 活动水平：1-久坐，2-轻度，3-中度，4-高度，5-极端
    Goal                TINYINT NOT NULL DEFAULT 0,  -- 目标：0-维持，1-减脂，2-增肌
    DailyCalorie        DECIMAL(18,2) NULL,       -- 自定义每日热量目标（NULL则用计算值）

    -- 账号状态
    Status              TINYINT NOT NULL DEFAULT 1,  -- 0-禁用，1-正常

    -- 时间戳
    CreatedAt           DATETIME2(3) NOT NULL DEFAULT GETDATE(),
    UpdatedAt           DATETIME2(3) NOT NULL DEFAULT GETDATE()
);
```

#### 索引

```sql
CREATE UNIQUE INDEX IX_Users_OpenId ON Users(OpenId) WHERE OpenId IS NOT NULL;
CREATE UNIQUE INDEX IX_Users_AppleId ON Users(AppleId) WHERE AppleId IS NOT NULL;
CREATE UNIQUE INDEX IX_Users_PhoneNumber ON Users(PhoneNumber) WHERE PhoneNumber IS NOT NULL;
CREATE INDEX IX_Users_UnionId ON Users(UnionId) WHERE UnionId IS NOT NULL;
```

#### 字段说明

| 字段 | 说明 |
|------|------|
| OpenId | 微信小程序用户唯一标识，微信分配 |
| UnionId | 同一微信开放平台下（公众号、小程序、APP），同一用户的唯一标识 |
| AppleId | Apple登录用户标识，由Apple提供 |
| PhoneNumber | 用户手机号，用于APP登录和账号关联 |
| PasswordHash | 密码哈希（用于后台管理等需要账号密码的场景） |
| VersionMode | 双版本模式：0-自律版，1-嘴馋版 |
| ActivityLevel | 活动水平，用于TDEE计算 |
| Goal | 健康目标，用于热量推荐调整 |
| DailyCalorie | 用户自定义每日热量目标，NULL时使用系统计算值 |

### 3.2 食物记录表（FoodRecords）

#### 表结构

```sql
CREATE TABLE FoodRecords (
    Id                  BIGINT PRIMARY KEY IDENTITY(1,1),
    UserId              INT NOT NULL,

    -- 记录信息
    RecordDate          DATE NOT NULL,            -- 记录日期
    MealType            TINYINT NOT NULL,          -- 餐次：1-早餐，2-午餐，3-晚餐，4-加餐

    -- 食物快照（冗余存储，避免历史数据因食物库变更而受影响）
    FoodId              BIGINT NULL,               -- 关联食物库ID（可为NULL，用户自定义食物）
    FoodName            NVARCHAR(100) NOT NULL,    -- 食物名称快照
    CategoryId          INT NULL,                  -- 分类ID快照
    CategoryName        NVARCHAR(255) NULL,        -- 分类名称快照

    -- 营养成分快照（记录时的值）
    Weight              DECIMAL(18,2) NOT NULL,   -- 重量(g)
    Energy              DECIMAL(18,2) NOT NULL,   -- 能量(kcal)
    Protein             DECIMAL(18,2) NOT NULL,   -- 蛋白质(g)
    Fat                 DECIMAL(18,2) NOT NULL,   -- 脂肪(g)
    Carbohydrate        DECIMAL(18,2) NOT NULL,   -- 碳水化合物(g)
    DietaryFiber        DECIMAL(18,2) NULL,       -- 膳食纤维(g)
    Cholesterol         DECIMAL(18,2) NULL,       -- 胆固醇(mg)
    -- 维生素快照（可选字段，根据食物库字段扩展）
    VitaminA            DECIMAL(18,2) NULL,
    VitaminC            DECIMAL(18,2) NULL,
    VitaminE            DECIMAL(18,2) NULL,
    -- 矿物质快照
    Calcium             DECIMAL(18,2) NULL,
    Iron                DECIMAL(18,2) NULL,
    Sodium              DECIMAL(18,2) NULL,
    Potassium           DECIMAL(18,2) NULL,
    Magnesium           DECIMAL(18,2) NULL,
    Zinc                DECIMAL(18,2) NULL,
    Phosphorus          DECIMAL(18,2) NULL,
    Selenium            DECIMAL(18,2) NULL,
    Copper              DECIMAL(18,2) NULL,
    Manganese           DECIMAL(18,2) NULL,
    Iodine              DECIMAL(18,2) NULL,

    -- 价格（可选）
    Price               DECIMAL(10,2) NULL,        -- 价格(元)

    -- 照片
    PhotoUrl            NVARCHAR(200) NULL,        -- 照片URL（云存储）
    PhotoLocalPath      NVARCHAR(500) NULL,       -- 本地照片路径（待AI识别用）

    -- 备注
    Remark              NVARCHAR(200) NULL,

    -- 时间戳
    CreatedAt           DATETIME2(3) NOT NULL DEFAULT GETDATE(),
    UpdatedAt           DATETIME2(3) NOT NULL DEFAULT GETDATE(),

    -- 外键
    CONSTRAINT FK_FoodRecords_Users FOREIGN KEY (UserId) REFERENCES Users(Id)
);
```

#### 索引

```sql
CREATE INDEX IX_FoodRecords_UserId_RecordDate ON FoodRecords(UserId, RecordDate DESC);
CREATE INDEX IX_FoodRecords_UserId_MealType ON FoodRecords(UserId, MealType);
CREATE INDEX IX_FoodRecords_FoodId ON FoodRecords(FoodId);
CREATE INDEX IX_FoodRecords_PhotoLocalPath ON FoodRecords(PhotoLocalPath) WHERE PhotoLocalPath IS NOT NULL;
```

#### 字段说明

| 字段 | 说明 |
|------|------|
| FoodId | 关联食物库中的食物ID，可为空（用户自定义食物时为空） |
| FoodName | 食物名称快照，冗余存储 |
| CategoryId/CategoryName | 分类信息快照 |
| Energy | **能量字段对应食物库的energy字段**，单位kcal |
| Weight | 用户输入的食用重量 |
| PhotoLocalPath | 本地照片路径，拍摄后暂存，为后续AI识别预留 |

## 四、多数据库配置

### 4.1 EF Core 多 DbContext

```csharp
// 食物库 DbContext（只读对接已有数据）
public class FoodLibraryDbContext : DbContext
{
    public DbSet<Food> Foods { get; set; }
    public DbSet<FoodCategory> FoodCategories { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    {
        optionsBuilder.UseSqlServer("Server=.;Database=QwTasteNote;Trusted_Connection=True;");
    }
}

// 业务库 DbContext
public class ChimeDbContext : DbContext
{
    public DbSet<User> Users { get; set; }
    public DbSet<FoodRecord> FoodRecords { get; set; }
    public DbSet<DailySummary> DailySummaries { get; set; }
    public DbSet<CustomFood> CustomFoods { get; set; }
    public DbSet<OperationLog> OperationLogs { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    {
        optionsBuilder.UseSqlServer("Server=.;Database=Chime;Trusted_Connection=True;");
    }
}
```

### 4.2 跨库查询示例

```csharp
// 查询食物及其分类（跨库）
public async Task<List<FoodWithCategory>> GetFoodsWithCategory(string keyword)
{
    using var foodContext = new FoodLibraryDbContext();

    var query = from f in foodContext.Foods
                join c in foodContext.FoodCategories on f.cate_id equals c.cate_id
                where f.food_name.Contains(keyword)
                select new FoodWithCategory
                {
                    Id = f.id,
                    FoodName = f.food_name,
                    CategoryName = c.title,
                    Energy = f.energy,
                    Protein = f.protein,
                    // ...
                };

    return await query.Take(20).ToListAsync();
}
```

## 五、数据同步与账号关联

### 5.1 多端账号关联策略

| 登录方式 | 标识字段 | 关联方式 |
|----------|----------|----------|
| 微信小程序 | OpenId | 直接创建账号 |
| iOS APP - 手机号 | PhoneNumber | 直接创建账号 |
| iOS APP - Apple | AppleId | 登录后需绑定手机号，再通过手机号关联 |

### 5.2 Apple登录绑定流程

```
Apple登录 → 获取AppleId → 检查是否已存在AppleId账号
    │
    ├─ 不存在 → 返回needBindPhone=true → 用户绑定手机号 → 创建/关联账号
    │
    └─ 存在 → 返回accessToken → 直接登录
```

### 5.3 账号关联查询

```sql
-- 通过手机号关联微信和Apple账号
SELECT * FROM Users
WHERE PhoneNumber = '13800138000';

-- 检查AppleId是否已存在
SELECT * FROM Users WHERE AppleId = 'AppleId123';

-- 检查OpenId是否已存在
SELECT * FROM Users WHERE OpenId = 'OpenId123';
```
